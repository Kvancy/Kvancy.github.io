1. GitHub:

github已经创建仓库后，将本地文件上传：

  1. 在本地文件夹中右键 Git Bash Here
  2. 输入 git init 在本地建立.git文件夹
  3. 输入 git remote add origin git@github.com:Kvancy/GameHall.git 设置远程仓库，origin后面地址从仓库SSH块复制
  4. 输入 git add .
  5. 输入 git commit -m "版本号"
  6. 输入 git checkout -b main 将上传分支设置为main(也可以是其他，自定义)
  7. 输入 git push -u origin main 上传文件到main分支

删除分支

  1. 远端分支 git push origin --delete [branch_name]
  2. 本地分支 git branch [branch_name] -d

查看分支

  1. 查看本地分支 git branch
  2. 查看远程分支 git branch -r
  3. 查看本地和远程分支 git branch -a

更换远程仓库地址

  1. git remote -v #查看远端地址
  2. git remote #查看远端仓库名
  3. git remote set-url origin https://gitee.com/xx/xx.git (新地址)

git push报错，显示仓库中有一部分代码，要先pull

参考链接

  • 方法一：

  1. git pull origin [main]分支名

  2. 重新add和commit相应文件

  3. git push origin [main] 分支名

  • 方法二：
  1. 强推： git push -f origin [main] 分支名

2. VScode的环境配置：

  1. launch.json文件配置（调试的配置，必须先编译得到exe）

  2. tasks.json文件配置（编译作用）,这里默认g++，不需要动，在终端-配置任务中修改

  3. 不知道为什么重启的时候launch.json文件总是刷新，重新配置下externalConsole和program就好，然后可以f5编译，右键run code 运行

3. 虚拟机配置：

bcdedit /set hypervisorlaunchtype off

4. 服务器搭建

1. 登录宝塔面板

  1. 远程连接云服务器的控制台

  2. 访问宝塔面板的安全入口，可通过控制台 su root bt default 查询网址，

    目前设置成了 47.97.68.254:1111/Kvancy

  3. 用户名是Kvancy

2. 本地下载hexo博客系统

  1. curl --silent --location https://rpm.nodesource.com/setup_14.x|sudo bash
  2. yum -y install nodejs
  3. npm install hexo-cli -g
  4. hexo init blog
  5. cd blog/
  6. npm install
  7. hexo server
  • 默认4000端口没用的话，使用 hexo server -p5000指定端口，如果想直接通过ip访问博客，可以通过 hexo server -p80设置端口。也可以通过 /blog/node_modules/hexo-server/index.js修改port值从而修改默认端口号
  • 有时候开启多个端口设置hexio会访问失败，通过 netstat -nultp查看所有端口号，kill -9 id id为进程id，杀死进程即可，kill - num 代表的意思如下
num 1 2 3 9 15 18 19 20
含义 进程重新被加载 删除进程在内存中的数据 删除鼠标在内存中的数据 强行结束单个进程 正常关闭进程 运行暂停的进程 暂停某个进程 把进程打入后台

3. 本地Hexo一键部署到远程服务器

参考链接

  1. yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec 安装宝塔面板

  2. yum install git 安装git

  3. adduser git 配置git用户

  4. chmod 740 /etc/sudoers 赋予用户权限 ``vi /etc/sudoers 进入权限管理文件

  5. root ALL=(ALL) ALL

    git ALL=(ALL) ALL 在这条上面加上这行

  6. chmod 400 /etc/sudoers 修改sudoers文件权限

  7. sudo passwd git 设置git 用户的密码，这里设置了 ‘root’

  8. su - git 给用户添加ssh密钥，git

    mkdir -p ~/.ssh

    touch ~/.ssh/authorized_keys

    chmod 600 ~/.ssh/authorzied_keys

    chmod 700 ~/.ssh vim ~/.ssh/authorized_keys 将ssh密钥粘贴进去，用于远程登录服务器

  9. ssh -v git@你的 ip 地址 输入密码 即可成功登录云服务器

  10. su root 创建一个git仓库，新建一个 post-receive 文件，用来存储本地的提交

    cd /home/git

    git init --bare blog.git #在/home/git下创建新仓库blog.git

    chown git:git -R blog.git #给予git用户权限

  11. cd blog.git/hooks vim post-receive 配置钩子文件，使得每次本地上传时，远程自动同步下载

  12. #!/bin/bash

`git --work-tree=/home/hexo --git-dir=/home/git/blog.git checkout -f`  写入钩子代码
  1. chmod +x /home/git/blog.git/hooks/post-receive 授予钩子可执行权限
以上是服务器配置

------

  1. deploy: #配置本地hexo的_config.yml文件
    type: git
    repo: root@你的ip地址:/home/git/blog.git    #仓库地址
    branch: master    #分支
